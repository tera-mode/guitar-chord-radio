# ギターコードラジオ — 要件定義書（自分用MVP）

## コンセプト

**「ギターを構えた30秒後には、もう弾いている」**

年代を選ぶとその時代のヒット曲がラジオのようにランダムで表示され、YouTubeで再生しながら横に表示されるコード譜を見て一緒に弾ける。曲を探す手間ゼロ、準備ゼロ。ギターを持ったらすぐ始められる体験を作る。

---

## ターゲット（将来的に）

- 30〜50代、学生時代にギターを触っていた層
- 上達よりも「弾く時間そのもの」に価値を感じる人
- ※ MVPはまず自分が毎日使いたいものを作る

---

## MVP機能一覧

### 1. 年代選択（トップ画面）

- 「80年代」「90年代」「2000年代」のボタンを並べる
- タップすると、その年代の曲リストからランダムで1曲を表示
- 将来的にジャンル（J-POP、洋楽ロック等）フィルターを追加

### 2. プレイヤー画面（メイン画面）

左右2カラム（モバイルは上下）のレイアウト。

**左側：YouTube再生エリア**
- YouTube IFrame Player APIで埋め込み再生
- 再生/停止ボタン
- 再生速度変更（0.5x / 0.75x / 1.0x）— ゆっくり弾きたい時用

**右側：コード譜エリア**
- 曲名・アーティスト名の表示
- コード進行をセクションごとに表示（イントロ / Aメロ / Bメロ / サビ 等）
- 各コード名をタップするとコードダイアグラム（押さえ方の図）をポップアップ表示
- ※ MVP段階では再生とコード譜の自動同期はしない（手動で目で追う）

**下部：ナビゲーション**
- 「次の曲」ボタン（同じ年代からランダムで次の曲へ）
- 「この曲をお気に入り」ボタン（ローカルストレージに保存、将来はFirebase）

### 3. コードダイアグラム表示

- 基本的なオープンコード（C, D, E, F, G, A, Am, Em, Dm 等）約20種のダイアグラムを用意
- SVGまたは画像で、押さえるフレット・弦を視覚的に表示
- コード名タップで表示/非表示を切り替え

### 4. 曲データ管理

Firestoreに格納する曲データの構造：

```json
{
  "id": "song_001",
  "title": "曲名",
  "artist": "アーティスト名",
  "year": 1992,
  "decade": "90s",
  "genre": "jpop",
  "youtubeId": "YouTube動画のID",
  "chords": {
    "intro": ["Am", "F", "G", "C"],
    "verse": ["Am", "F", "G", "C", "Am", "F", "G", "E"],
    "chorus": ["F", "G", "Am", "C", "F", "G", "C"],
    "outro": ["Am", "F", "G", "C"]
  },
  "capo": 0,
  "difficulty": "easy",
  "tags": ["定番", "弾き語り"]
}
```

### 5. 初期コンテンツ（自分で登録する曲リスト）

MVPでは **30曲** を目標に手動登録する。以下は選曲の方針：

- 80年代：10曲（尾崎豊、BOØWY、サザン 等）
- 90年代：10曲（スピッツ、Mr.Children、GLAY 等）
- 2000年代：10曲（BUMP OF CHICKEN、レミオロメン、MONGOL800 等）
- 選曲基準：コード進行がシンプル（5〜7種以内）、YouTubeに公式MVまたは公式オーディオがある、自分が弾きたい曲

---

## MVPに含めないもの（将来検討）

| 機能 | 理由 |
|------|------|
| コードと再生の自動同期 | タイムスタンプ作成の工数が大きい |
| AI自動コード検出 | 精度検証が必要、後回し |
| ユーザー認証・ログイン | 自分用なので不要 |
| お気に入り永続化（DB） | まずはローカルストレージで十分 |
| かんたんアレンジ版コード | 曲数が増えてから対応 |
| ソーシャル機能 | 個人利用フェーズでは不要 |
| モバイルアプリ化 | Webアプリで十分、PWA化は検討 |

## 開発ステップ

### Step 1：骨組み（1〜2日）
- Next.jsプロジェクト作成、Tailwind設定、Vercelデプロイ
- トップ画面のUI（年代選択ボタン）
- プレイヤー画面のレイアウト（2カラム）

### Step 2：YouTube再生（1日）
- YouTube IFrame Player API組み込み
- 再生/停止/速度変更の実装

### Step 3：コード譜表示（1〜2日）
- Firestoreに曲データ投入（まず5曲程度）
- コード進行のセクション別表示
- コードダイアグラムのポップアップ

### Step 4：ランダム選曲とナビ（1日）
- 年代指定でランダム1曲取得
- 「次の曲」ボタン
- お気に入りボタン（ローカルストレージ）

### Step 5：コンテンツ充実（継続的）
- 30曲分のコード譜データを作成・登録
- 各曲のYouTube動画IDを確認・設定

### Step 6：仕上げ（1〜2日）
- レスポンシブ対応（スマホでも使いやすく）
- OGP・ファビコン等
- PWA化の検討

**想定開発期間：約1〜2週間（コンテンツ登録含む）**

---

## 補足：著作権まわりの整理

- **YouTube埋め込み再生**：YouTube利用規約に準拠した埋め込みは問題なし
- **コード譜の掲載**：コード進行そのものには著作権はないとされている（メロディ・歌詞には著作権あり）。コード進行のみを掲載し、歌詞は一切載せない方針
- **個人利用**：MVPは自分用・非公開なのでリスクは低い。公開する際はJASRAC許諾の要否を確認する
