# ギターコードラジオ — リブランディング & UI洗練 要件定義

## 概要

以下3つの作業をまとめて行う。

1. サービス名・プロジェクト名・ドメイン関連の総入れ替え
2. UI/CXのぜい肉削減（トップ画面・プレイヤー画面）
3. 「ギター弾きたい→即起動→即弾ける」の想起性・起動性強化

## 全体ルール

- 報告は必ず日本語
- 管理者の許可なく `git push` / Vercel デプロイは実行しない
- 既存機能の挙動を壊す変更は事前確認
- 1要件ずつ着手し、完了ごとに動作確認の報告
- 既存のスタイル方針（Tailwind, amber系カラー）を踏襲

## 要件2: UIのぜい肉削減

### 設計原則

> **「機能を残し、説明を消す」**
> アイコン・配置・年号タイポで意味が通るなら、ラベルは要らない。

### 2-1. トップ画面 (`app/page.tsx`)

**削除するもの**

- 中央の `🎸` 絵文字（タイトルの上）
- 「年代を選ぶ」セクション見出し（カードが並んでいれば自明）
- 各年代カード内のアーティスト名サブテキスト（「尾崎豊 / BOØWY / サザン」等）
- 各年代カード内の絵文字（🎸 🎵 🎶 — 識別性に寄与しない）
- お気に入りカードのサブテキスト「保存した曲を聴く」（❤️ アイコンで自明）
- フッターの「コード名をタップすると押さえ方が見られます」

**残す・強化するもの**

- アプリ名 `ギターコードラジオ`
- 新キャッチコピー
- 年代カード3つ＋お気に入りカード1つ
- **年号を主役に**: カード内の「80年代」をより大きく（`text-xl` → `text-2xl` 程度）、`80s / 90s / 2000s` のような英数字併記もアリ

**追加するもの（要件4と関連）**

- 最後に聴いた曲がある場合、年代カードの**上**に「続きから」カードを1枚表示

### 2-2. プレイヤー画面ヘッダー (`PlayerClient.tsx`)

**削除するもの**

- 戻るリンクの「年代選択」テキスト（`←` アイコンのみ残す）
- 中央の `🎸 80年代` / `❤️ お気に入り` ラベル

**残すもの**

- `←` アイコン（ホームへ戻る）
- ヘッダー自体の sticky 挙動

### 2-3. プレイヤー画面コード譜エリア (`ChordSheet.tsx`)

**削除するもの**

- セクション見出し「使用コード一覧」（チャンクが並んでいれば自明）
- セクション見出し「コード進行」相当の表示（あれば）
- 難易度バッジ（初級/中級/上級 — ランダム再生主体では判断材料にならない）
- タグ表示（`#定番 #弾き語り` — 機能している瞬間がない）
- お気に入りボタンの「お気に入り」テキストラベル（PCレイアウトも含めアイコンのみに統一）

**残すもの**

- 曲タイトル / アーティスト名 / 年
- カポ表記（`♪2` のコンパクト形式に統一、PCも同じ）
- 使用コード一覧のダイアグラム並び
- セクション別コード進行（`イントロ` `Aメロ` `サビ` 等のラベルは残す — これは情報として必要）
- お気に入りボタン（アイコンのみ）

### 2-4. プレイヤー画面コントロール (`YouTubePlayer.tsx`)

**削除するもの**

- 「速度」ラベル（数字ボタンが並んでいれば自明）

**残すもの**

- 再生/停止ボタン
- 速度切替ボタン（`0.5x` / `0.75x` / `1.0x`）

### 2-5. プレイヤー画面フッター

**削除するもの**

- 再生モードボタン下の「オフ / 自動 / リピート」テキストラベル（アイコンのみに統一）

**残すもの**

- 再生モードアイコン（`⏹` / `⏭` / `🔂`）
- 「次の曲」ボタン

## 要件3: 「ギター弾きたい→即弾ける」の起動性強化

### 3-1. 自動再生（最重要）

ホームから年代タップ後、プレイヤー画面で**動画を自動再生開始**する。

- YouTube IFrame Player API は音声付き自動再生がブラウザ仕様で禁止されている
- 解決策: `mute=1` パラメータ付きで自動再生を開始し、ユーザーの再生ボタンタップで `unMute()` を呼ぶ
- ミュート中は再生ボタンに小さく `🔇` を重ねて「タップで音を出す」ことを暗示
- ヒントテキスト「タップで音を出す」を**初回のみ**ボタン下に小さく表示（localStorage `gcr-hint-unmute-shown` で制御）

### 3-2. 「続きから」機能

ホーム画面の年代カードの**上**に、最後に聴いた曲を再生再開できるカードを1枚配置。

- 曲が変わるたびに `gcr-last-song` キーで `{ songId, source }` を localStorage に保存
- ホーム読み込み時に上記が存在すれば「続きから」カードを表示
- カード内容: `▶ 続きから` / 曲タイトル・アーティスト（小さく）
- カードタップで `/player?source={source}&songId={songId}` に遷移
- `PlayerClient.tsx` 側で `songId` クエリがあればその曲から開始

### 3-3. フローティング「次の曲」ボタン（モバイルのみ）

ギターを構えたまま親指1本で押せる位置に配置。

- モバイル時のみ画面右下に円形フローティングボタン（`fixed bottom-4 right-4`）
- 内容: `⏭` アイコン
- 既存のフッター「次の曲」ボタンはPCレイアウトでのみ表示
- フッター自体は再生モードボタン用に残す（モバイルでもフッター中央に配置）

### 3-4. お気に入り追加時のマイクロアニメーション

ハートをタップした瞬間、`scale-up → scale-back` の一瞬の弾みアニメーション。

- Tailwind の `transition-transform` + `active:scale-125` 程度の軽い実装でよい
- 過度な装飾は不要

### 3-5. PWA化（ホーム画面追加対応）

「弾きたい→アイコンタップで即起動」を実現するため、PWA最低要件を満たす。

- `manifest.json` を作成
  - `name`: ギターコードラジオ
  - `short_name`: ギターコードラジオ
  - `start_url`: `/`
  - `display`: `standalone`
  - `background_color` / `theme_color`: amber 系
  - `icons`: 192x192 / 512x512（後日差し替え予定としてプレースホルダーでOK）
- 初回訪問時の「ホーム画面に追加」案内は今回はスコープ外

## 実装順序の推奨

依存関係を考慮し、以下の順で進めること。

1. **要件1: 名称・プロジェクト構成変更** — 全体の基盤。先にやらないと後の作業が無駄になる
   - 1-1 フォルダリネーム → 1-2 package.json → 1-5 localStorage キー → 1-3/1-4 表示文字列 → 1-6 ドキュメント
2. **要件2: UIぜい肉削減** — 既存挙動を壊さない見た目の整理
3. **要件3: 起動性強化** — 機能追加を最後に
   - 3-1 自動再生 → 3-2 続きから → 3-3 フローティング → 3-4 アニメ → 3-5 PWA

各要件完了時に以下を報告:
- 変更したファイル一覧
- 動作確認した内容（モバイル/PC両方）
- 既存機能への影響有無
